MySQL CASE Statement

MySQL இல் CASE அறிக்கையைப் பயன்படுத்தி நிபந்தனைகளைச் சோதித்து மதிப்புகளை வழங்க கற்றுக்கொள்ளுங்கள்

MySQL CASE அறிக்கை

CASE அறிக்கை நிபந்தனைகளைக் கடந்து, முதல் நிபந்தனை நிறைவேற்றப்படும் போது ஒரு மதிப்பை வழங்குகிறது (if-then-else அறிக்கை போன்றது). எனவே, ஒரு நிபந்தனை உண்மையாகும்போது, அது வாசிப்பதை நிறுத்தி முடிவை வழங்கும். எந்த நிபந்தனைகளும் உண்மையாக இல்லாவிட்டால், அது ELSE பகுதியில் உள்ள மதிப்பை வழங்கும்.

ELSE பகுதி இல்லை மற்றும் எந்த நிபந்தனைகளும் உண்மையாக இல்லாவிட்டால், அது NULL ஐ வழங்கும்.

CASE தொடரியல்

CASE
    WHEN condition1 THEN result1
    WHEN condition2 THEN result2
    WHEN conditionN THEN resultN
    ELSE result
END;

MySQL CASE எடுத்துக்காட்டுகள்

பின்வரும் SQL நிபந்தனைகளைக் கடந்து, முதல் நிபந்தனை நிறைவேற்றப்படும் போது ஒரு மதிப்பை வழங்குகிறது:

SELECT OrderID, Quantity,
CASE
    WHEN Quantity > 30 THEN 'The quantity is greater than 30'
    WHEN Quantity = 30 THEN 'The quantity is 30'
    ELSE 'The quantity is under 30'
END AS QuantityText
FROM OrderDetails;

பின்வரும் SQL வாடிக்கையாளர்களை City மூலம் வரிசைப்படுத்தும். இருப்பினும், City NULL ஆக இருந்தால், Country மூலம் வரிசைப்படுத்தும்:

SELECT CustomerName, City, Country
FROM Customers
ORDER BY
(CASE
    WHEN City IS NULL THEN Country
    ELSE City
END);

பயிற்சி

❓ SQL CASE வெளிப்பாட்டை முடிக்க சரியான முக்கிய சொற்களைத் தேர்ந்தெடுக்கவும்.

CASE
✓ சரி! CASE என்பது CASE வெளிப்பாட்டைத் தொடங்கும் முக்கிய சொல்லாகும்
WHEN Quantity > 50 THEN 'High'
✓ சரி! இது 50க்கு மேல் அளவு உள்ள போது 'High' என வழங்கும் நிபந்தனையாகும்
WHEN Quantity <= 50 THEN 'Low'
✓ சரி! இது 50 அல்லது அதற்கும் குறைவான அளவு உள்ள போது 'Low' என வழங்கும் நிபந்தனையாகும்
ELSE
✓ சரி! ELSE என்பது மற்ற எல்லா நிகழ்வுகளுக்கும் மதிப்பை வழங்கப் பயன்படுகிறது
'Unknown'
✓ சரி! இது ELSE பகுதிக்கான மதிப்பாகும்
END
✓ சரி! END என்பது CASE வெளிப்பாட்டை முடிக்கும் முக்கிய சொல்லாகும்
AS QuantityCategory
✓ சரி! இது வெளியீட்டு நெடுவரிசைக்கு ஒரு மாற்றுப் பெயரை வழங்குகிறது
