MySQL LIMIT விதி

MySQL LIMIT விதி

LIMIT விதி திரும்பப் பெற வேண்டிய பதிவுகளின் எண்ணிக்கையைக் குறிப்பிட பயன்படுகிறது.

ஆயிரக்கணக்கான பதிவுகளைக் கொண்ட பெரிய அட்டவணைகளில் LIMIT விதி பயனுள்ளதாக இருக்கிறது. பெரிய எண்ணிக்கையிலான பதிவுகளைத் திருப்பித் தருவது செயல்திறனைப் பாதிக்கும்.

LIMIT தொடரமைப்பு

SELECT column_name(s)
FROM table_name
WHERE condition
LIMIT number;

MySQL LIMIT எடுத்துக்காட்டுகள்

பின்வரும் SQL அறிக்கை "Customers" அட்டவணையில் இருந்து முதல் மூன்று பதிவுகளைத் தேர்ந்தெடுக்கிறது:

எடுத்துக்காட்டு: உங்கள் சொந்த SQL சர்வரைப் பெறுங்கள்

SELECT * FROM Customers
LIMIT 3;

பதிவுகள் 4 - 6 (சேர்க்கை) ஐத் தேர்ந்தெடுக்க விரும்பினால் என்ன செய்வது?

MySQL இதைக் கையாள ஒரு வழியை வழங்குகிறது: OFFSET ஐப் பயன்படுத்துவதன் மூலம்.

கீழே உள்ள SQL கோரிக்கை "3 பதிவுகளை மட்டும் திருப்பித் தரவும், பதிவு 4 இல் தொடங்கவும் (OFFSET 3)" என்று கூறுகிறது:

எடுத்துக்காட்டு

SELECT * FROM Customers
LIMIT 3 OFFSET 3;

WHERE விதியைச் சேர்க்கவும்

பின்வரும் SQL அறிக்கை country "Germany" ஆக உள்ள "Customers" அட்டவணையில் இருந்து முதல் மூன்று பதிவுகளைத் தேர்ந்தெடுக்கிறது:

எடுத்துக்காட்டு

SELECT * FROM Customers
WHERE Country='Germany'
LIMIT 3;

ORDER BY விதியைச் சேர்க்கவும்

பின்வரும் SQL அறிக்கை தேர்விலிருந்து முதல் மூன்று பதிவுகளைத் திருப்பித் தருவதற்கு முன் வாடிக்கையாளர்களை country மூலம் வரிசைப்படுத்துகிறது:

எடுத்துக்காட்டு

SELECT * FROM Customers
ORDER BY Country
LIMIT 3;

பயிற்சி

❓ பின்வருவனவற்றில் எது சரியான SQL அறிக்கையாகும்:

SELECT * FROM Customers LIMIT 5
✓ சரி! இது சரியான SQL தொடரமைப்பாகும் - SELECT அறிக்கையின் முடிவில் LIMIT விதி வருகிறது
LIMIT 5;SELECT * FROM Customers
✗ தவறு! LIMIT விதி SELECT அறிக்கையின் முடிவில் மட்டுமே வர முடியும், தொடக்கத்தில் அல்ல
SELECT LIMT 5 * FROM Customers
✗ தவறு! இது தவறான தொடரமைப்பு. "LIMIT" என்று எழுதப்பட்டுள்ளது (LIMT அல்ல) மற்றும் அது SELECT அறிக்கையின் முடிவில் வர வேண்டும்
